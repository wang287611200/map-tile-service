## 背景与目标

车载地图数据具有多区域、多版本、多引擎兼容等复杂管理需求。本方案旨在建立一套完整的多版本数据管理体系，支撑以下6个核心目标：

1. **增量更新**：仅下载变化的瓦片或区域数据，降低流量消耗
2. **版本回退**：当新版本存在问题时，能快速回滚至上一稳定版本
3. **灰度发布**：按车辆 ID 分桶，逐步扩大新版本覆盖范围
4. **引擎兼容**：地图渲染引擎版本与数据格式版本存在兼容约束，服务端统一决策
5. **离线/在线一致性**：预下载的离线包与在线实时瓦片使用同一份数据，行为一致
6. **数据新鲜度**：确保车端地图数据在可接受延迟内保持更新

---

## 业务场景

### 场景 1：实时瓦片下载
车端在导航过程中动态请求当前视口所需的地图瓦片。要求低延迟、高可用，支持 CDN 加速。

### 场景 2：离线包分区下载
车端在 WiFi 或低成本网络环境下，预先下载特定行政区划的完整离线包，在无网络时使用。
