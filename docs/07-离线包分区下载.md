# 第7章 离线包分区下载

---

## 9.1 分区+版本矩阵

不同区域的数据版本可以处于不同状态，以下是典型的版本矩阵示例：

| 区域 | v20250101 | v20250601 | v20260101 |
|------|-----------|-----------|-----------|
| CN_NORTH | DEPRECATED | PUBLISHED | CANARY |
| CN_EAST | DEPRECATED | PUBLISHED | PUBLISHED |
| CN_SOUTH | DEPRECATED | DEPRECATED | PUBLISHED |
| CN_WEST | DEPRECATED | PUBLISHED | DRAFT |

车端查询离线包时，系统根据车辆的引擎版本和分桶，返回各区域最合适的可下载版本。

### 分区版本查询接口

```yaml
GET /api/v1/regions/{region_id}/versions
Query Parameters:
  engine_version: "3.5.0"             # 必填，用于过滤兼容版本
  data_type: "road_network"            # 可选，默认返回所有类型

Response (200 OK):
  region_id: "CN_NORTH"
  available_versions:
    - version_no: "v20260101"
      format_version: "format_v4.0"
      status: "CANARY"                 # 该引擎版本分桶内可见
      packages:
        - package_type: "full"
          package_url: "https://cdn.example.com/offline/CN_NORTH/v20260101/full.pkg"
          package_size: 21474836480    # 20GB
          checksum_sha256: "a3f9b2..."
          supports_resume: true        # 支持断点续传
        - package_type: "incremental"
          from_version: "v20250601"
          package_url: "https://cdn.example.com/offline/CN_NORTH/v20260101/incr_from_v20250601.pkg"
          package_size: 1073741824     # 1GB
          checksum_sha256: "b4c5d6..."
          supports_resume: true
    - version_no: "v20250601"
      status: "PUBLISHED"
      packages:
        - package_type: "full"
          package_url: "..."
          package_size: 20000000000
```

### 离线包下载接口

```yaml
GET /download/offline/{package_id}

Request Headers:
  Range: "bytes=0-1073741823"          # 支持 Range 请求（断点续传）
  Authorization: "Bearer <token>"

Response (206 Partial Content):
  Headers:
    Content-Range: "bytes 0-1073741823/21474836480"
    Content-Type: "application/octet-stream"
    ETag: "\"sha256:a3f9b2...\""
    Accept-Ranges: "bytes"
    Cache-Control: "public, max-age=2592000, immutable"
```

### 状态上报接口

```yaml
POST /api/v1/vehicles/{vehicle_id}/status

  region_id: "CN_NORTH"
```

## 9.3 离线包存储结构（OSS 目录）

```
oss://offline-packages/
└── road_network/
    ├── CN_NORTH/
    │   ├── v20250601/
    │   │   ├── full.pkg                       # 全量包（~20GB）
    │   │   └── metadata.json                  # 包元数据
    │   └── v20260101/
    │       ├── full.pkg                       # 全量包（~20GB）
    │       ├── incr_from_v20250601.pkg        # 增量包（~1GB）
    │       ├── incr_from_v20250101.pkg        # 更大的增量包（~3GB）
    │       └── metadata.json
    ├── CN_EAST/
    │   └── v20260101/
    │       ├── full.pkg
    │       └── incr_from_v20250601.pkg
    └── CN_SOUTH/
        └── v20260101/
            └── full.pkg
```

### 离线包内部结构

每个 `.pkg` 文件为标准 ZIP 格式，内部结构如下：

```
full.pkg (ZIP)
├── header.json                        # 包元数据
│   {
│     "version_no": "v20260101",
│     "region_id": "CN_NORTH",
│     "data_type": "road_network",
│     "format_version": "format_v4.0",
│     "tile_count": 1580320,
│     "created_at": "2026-01-10T06:00:00Z"
│   }
├── tiles/                             # 瓦片文件（按 z/x/y 组织）
│   ├── 3/
│   │   └── 6/
│   │       └── 3.pbf
│   ├── 14/
│   │   └── 13708/
│   │       ├── 6411.pbf
│   │       └── 6412.pbf
│   └── ...
└── tile_index.db                      # SQLite 瓦片索引（加速查找）
```

## 9.4 增量包生成流程

```
旧版本 v20250601          新版本 v20260101
  tile_index                tile_index
      │                         │
      └──────────┬──────────────┘
                 │
                 ▼
          ┌────────────┐
          │ Diff Engine │
          │ (对比两版本  │
          │  的瓦片索引) │
          └─────┬───────┘
                │
      ┌─────────┼─────────┐
      ▼         ▼         ▼
   新增瓦片   修改瓦片   删除瓦片
   (added/)  (modified/) (deleted.list)
      │         │         │
      └─────────┼─────────┘
                │
                ▼
        ┌──────────────┐
        │  增量包输出   │
        │incr_from_     │
        │v20250601.pkg  │
        │               │
        │ header.json   │
        │ added/        │
        │ modified/     │
        │ deleted.list  │
        └──────────────┘
```

### 增量包 header.json 示例

```json
{
  "package_type": "incremental",
  "from_version": "v20250601",
  "to_version": "v20260101",
  "region_id": "CN_NORTH",
  "data_type": "road_network",
  "stats": {
    "added_tiles": 12300,
    "modified_tiles": 45600,
    "deleted_tiles": 8900,
    "total_changes": 66800
  },
  "apply_instructions": "先应用 deleted.list 删除，再写入 added/ 和 modified/ 目录"
}
```

### 车端应用增量包的流程

```python
def apply_incremental_package(pkg_path: str, cache_dir: str):
    pkg = ZipFile(pkg_path)
    header = json.loads(pkg.read('header.json'))

    # Step 1: 删除已删除的瓦片
    for tile_key in pkg.read('deleted.list').decode().splitlines():
        os.remove(os.path.join(cache_dir, tile_key))

    # Step 2: 写入新增瓦片
    for name in pkg.namelist():
        if name.startswith('added/'):
            target = os.path.join(cache_dir, name[6:])  # 去掉 'added/' 前缀
            os.makedirs(os.path.dirname(target), exist_ok=True)
            with open(target, 'wb') as f:
                f.write(pkg.read(name))

    # Step 3: 覆盖修改的瓦片
    for name in pkg.namelist():
        if name.startswith('modified/'):
            target = os.path.join(cache_dir, name[9:])  # 去掉 'modified/' 前缀
            with open(target, 'wb') as f:
                f.write(pkg.read(name))

    # Step 4: 更新本地版本记录
    update_local_version(header['region_id'], header['to_version'])
```
