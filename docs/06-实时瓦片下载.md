## 8.1 请求链路

```
┌──────────────┐        ┌──────────────┐        ┌──────────────┐
│   车端        │        │     CDN      │        │     OSS      │
│              │        │   边缘节点   │        │  源站存储    │
│  地图渲染    │──①──>  │              │──③──>  │              │
│  引擎        │        │  ②缓存命中   │        │ map/v1024/   │
│              │<──②──  │  直接返回    │        │ {z}/{x}/{y}  │
│              │        │  ④缓存未命中 │<──④──  │ .pbf         │
│              │<──④──  │  回源后缓存  │        │              │
└──────────────┘        └──────────────┘        └──────────────┘

① 车端请求: GET /tiles/road_network/v20260101/{z}/{x}/{y}.pbf
② 若 CDN 有缓存且未过期，直接返回（命中率 >95%）
③ CDN 缓存未命中，向 OSS 回源
④ OSS 返回瓦片文件，CDN 缓存后返回给车端
```

## 8.2 API 设计

### 版本协商接口

```yaml
POST /api/v1/versions/check

Request Body:
  vehicle_id: "VH_ABC123"               # 车辆唯一标识
  engine_version: "3.5.0"              # 当前渲染引擎版本
  supported_features:                   # 引擎支持的特性列表
    - "3d_building"
    - "hd_road"
    - "vector_poi"
  current_versions:                     # 各区域当前使用的版本
    CN_BJ: "v20250601"
    CN_TJ: "v20250601"
    CN_NORTH: "v20250601"

Response Body (200 OK):
  CN_BJ:
    latest_compatible_version: "v20260101"
    format_version: "format_v4.0"
    compatibility: "FULL"
    tile_url_template: "https://cdn.example.com/tiles/road_network/v20260101/{z}/{x}/{y}.pbf"
    cdn_token: "eyJhbGc..."              # 临时鉴权 token（可选）
    action_required: "UPDATE"            # UPDATE / NO_UPDATE / ENGINE_UPGRADE_REQUIRED
  CN_TJ:
    latest_compatible_version: "v20260101"
    action_required: "UPDATE"
  CN_NORTH:
    latest_compatible_version: "v20260101"
    action_required: "UPDATE"
```

### 版本清单接口

```yaml
GET /api/v1/manifest
Query Parameters:
  region_id: "CN_NORTH"                 # 可选，不传则返回全国
  data_type: "road_network"             # 可选

Request Headers:
  If-None-Match: "sha256:prev_hash..."  # 条件请求，避免重复拉取

Response (200 OK - 有更新):
  Headers:
    ETag: "sha256:new_hash..."
    Cache-Control: "public, max-age=300"  # 5分钟
  Body:
    region_versions:
      CN_NORTH: "v20260101"
      CN_EAST: "v20260101"
      CN_SOUTH: "v20250601"
    manifest_hash: "sha256:a3f9b2c1..."
    generated_at: "2026-03-10T08:00:00Z"

Response (304 Not Modified - 无更新):
  Headers:
    ETag: "sha256:prev_hash..."
  Body: (empty)
```

### 瓦片下载接口

```yaml
GET /tiles/{data_type}/{version_no}/{z}/{x}/{y}.pbf

Path Parameters:
  data_type: "road_network"
  version_no: "v20260101"
  z: 14                   # Zoom 级别
  x: 13708               # 瓦片 X 坐标
  y: 6411                # 瓦片 Y 坐标

Response Headers:
  Content-Type: "application/x-protobuf"
  ETag: "\"sha256:a3f9b2c1...\""
  Cache-Control: "public, max-age=2592000, immutable"  # 30天，不可变
  Content-Length: 45678
```

> **关键设计**：版本号编码在 URL 路径中（`/v20260101/`），使得同一版本的瓦片 URL 永久不变，
> CDN 可以无限期缓存（`immutable`）。版本变化通过版本协商发现新 URL，而非缓存失效。

## 8.3 版本协商完整查询逻辑（Python 伪代码）

```python
def get_tile_url_for_vehicle(vehicle_id: str, engine_version: str,
                              region_id: str, data_type: str) -> dict:

    # Step 1: 查询该引擎兼容的数据格式版本列表
    compatible_formats = db.query("""
        SELECT format_version
        FROM compatibility_matrix
        WHERE engine_version = :engine_version
          AND compatibility_level != 'INCOMPATIBLE'
        ORDER BY format_version DESC
    """, engine_version=engine_version)

    # Step 2: 逐区域查询最新兼容版本
    candidates = db.query("""
        SELECT v.version_id, v.version_no, v.format_version, vr.status
        FROM versions v
        JOIN version_regions vr ON v.version_id = vr.version_id
        WHERE vr.region_id = :region_id
          AND v.data_type = :data_type
          AND v.format_version = ANY(:formats)
          AND vr.status IN ('PUBLISHED', 'CANARY')
          AND v.min_engine_version <= :engine_version
        ORDER BY v.version_no DESC
        LIMIT 10
    """, region_id=region_id, data_type=data_type,
         formats=compatible_formats, engine_version=engine_version)

    # Step 3: 灰度判断（与第5章灰度判断逻辑一致）
    selected = None
    for c in candidates:
        if c.status == 'PUBLISHED':
            selected = c
            break
        elif c.status == 'CANARY' and is_in_canary(vehicle_id, c.version_id, region_id):
            selected = c
            break

    # Step 4: 检查是否有不兼容的更新版本（提示引擎升级）
    engine_upgrade_hint = check_newer_incompatible_version(
        region_id, data_type, selected.version_no, engine_version
    )

    return {
        "version_no": selected.version_no,
        "tile_url_template": f"https://cdn.example.com/tiles/{data_type}/{selected.version_no}/{{z}}/{{x}}/{{y}}.pbf",
        "engine_upgrade_hint": engine_upgrade_hint,
    }
```

    │   ├── CN_NORTH/

    │   │   │   │   ├── 6411.pbf
    │   │   │   │   └── 6412.pbf
    │   │   │   └── 13709/

        ├── CN_NORTH/
        │   ├── 14/
        │   │   └── 13708/
        │   │       ├── 6411.pbf      # 实际存储（该瓦片有变化）

路径规范：`map/{data_type}/v{version_no}/{region_id}/{z}/{x}/{y}.pbf`

## 8.5 Copy-on-Write（CoW）存储优化

版本更新时，大量瓦片数据实际上没有发生变化（如偏远地区的背景图层）。
通过 CoW 机制，只存储真正变化的瓦片，大幅节省存储空间：

```
新版本 v20260101 与旧版本 v20250601 的瓦片差异分析：

┌──────────────────────────────────────────────┐
│  瓦片分类           数量        处理方式       │
├──────────────────────────────────────────────┤
│  有变化的瓦片       ~5%         实际存储新文件 │
│  未变化的瓦片       ~95%        storage_key   │
│                                 指向旧版本路径 │
└──────────────────────────────────────────────┘

存储节省：~95%
```

### CoW 实现方式

```sql
-- 有变化的瓦片：存新路径
INSERT INTO tile_index (version_id, region_id, data_type, zoom_level, tile_x, tile_y, storage_key)
VALUES (1024, 'CN_NORTH', 'road_network', 14, 13708, 6411,
        'road_network/v20260101/CN_NORTH/14/13708/6411.pbf');  -- 新路径

-- 未变化的瓦片：指向旧版本路径
INSERT INTO tile_index (version_id, region_id, data_type, zoom_level, tile_x, tile_y, storage_key)
VALUES (1024, 'CN_NORTH', 'road_network', 14, 13708, 6412,
        'road_network/v20250601/CN_NORTH/14/13708/6412.pbf');  -- 旧版本路径（CoW）
```

### 版本清理时的注意事项

清理旧版本时，需先检查是否有其他版本的 `tile_index.storage_key` 引用了该版本的路径，
确保无引用后再删除 OSS 文件，避免"悬空引用"。
