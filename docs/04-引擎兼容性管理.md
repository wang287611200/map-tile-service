## 6.1 兼容性约束模型

### 双向约束规则

```
约束1（引擎下界）：version.min_engine_version ≤ vehicle.engine_version
约束2（格式上界）：engine.min_data_format ≤ version.format_version ≤ engine.max_data_format
```

只有同时满足以上两个约束，车辆才可以使用该版本数据。

### 典型不兼容场景

| 场景 | 描述 | 后果 |
|------|------|------|
| 引擎过旧 | `vehicle.engine_version < version.min_engine_version` | 引擎无法解析新格式字段，渲染崩溃或数据错误 |
| 引擎过新 | `version.format_version < engine.min_data_format` | 引擎不再支持旧格式，无法解析 |
| 格式超出引擎支持范围 | `version.format_version > engine.max_data_format` | 引擎不认识新格式，忽略关键字段 |
| 格式为 Breaking Change | `format.breaking_change = TRUE` 但引擎未更新 | 引擎按旧协议解析新格式，数据语义错误 |

### 流程3：灰度准入校验

**引擎 3.2.0（不通过）**：
```
canary_ratio = 20%  (前20%的分桶可用)
hash("VH_ABC123") % 100 = 67  → 67 > 20 → 不进入灰度
→ 返回上一个 PUBLISHED 版本
```

**引擎 3.5.0（通过）**：
```
canary_ratio = 20%
hash("VH_XYZ789") % 100 = 12  → 12 ≤ 20 → 进入灰度
→ 同时检查 compatibility_matrix: 3.5.0 × format_v4.1 = FULL
→ 返回灰度版本 v20260101
```

### 流程4：引擎OTA升级安全校验


## 版本协商完整查询逻辑（Python 伪代码）

```python
def version_check(vehicle_id: str, engine_version: str,
                  region_id: str, current_version: str) -> dict:

    # Step 1: 查询该引擎兼容的数据格式版本列表
    compatible_formats = db.query("""
        SELECT format_version, compatibility_level
        FROM compatibility_matrix
        WHERE engine_version = :engine_version
          AND compatibility_level != 'INCOMPATIBLE'
        ORDER BY format_version DESC
    """, engine_version=engine_version)

    format_list = [r.format_version for r in compatible_formats]

    # Step 2: 逐区域查询最新兼容版本（考虑 PUBLISHED 和 CANARY 状态）
    candidates = db.query("""
        SELECT v.version_id, v.version_no, v.format_version,
               vr.status, v.min_engine_version
        FROM versions v
        JOIN version_regions vr ON v.version_id = vr.version_id
        WHERE vr.region_id = :region_id
          AND v.format_version = ANY(:format_list)
          AND vr.status IN ('PUBLISHED', 'CANARY')
          AND v.min_engine_version <= :engine_version
        ORDER BY v.version_no DESC
        LIMIT 10
    """, region_id=region_id, format_list=format_list,
         engine_version=engine_version)

    # Step 3: 灰度准入判断
    latest = None
    for candidate in candidates:
        if candidate.status == 'PUBLISHED':
            latest = candidate
            break
        elif candidate.status == 'CANARY':
            # 检查车辆是否在灰度分桶内
            canary_config = get_canary_config(candidate.version_id, region_id)
            bucket = hash(vehicle_id) % 100
            if bucket < canary_config.ratio:
                latest = candidate
                break

    if latest is None:
        return {"action": "NO_UPDATE", "current_version": current_version}

    # Step 4: 检查是否存在不兼容的更新版本（提示引擎升级）
    newer_incompatible = db.query("""
        SELECT v.version_no, v.min_engine_version
        FROM versions v
        JOIN version_regions vr ON v.version_id = vr.version_id
        WHERE vr.region_id = :region_id
          AND v.version_no > :latest_version
          AND v.min_engine_version > :engine_version
          AND vr.status = 'PUBLISHED'
        ORDER BY v.version_no DESC
        LIMIT 1
    """, region_id=region_id, latest_version=latest.version_no,
         engine_version=engine_version)

    result = {
        "latest_version": latest.version_no,
        "format_version": latest.format_version,
        "action": "UPDATE" if latest.version_no != current_version else "NO_UPDATE",
    }
    if newer_incompatible:
        result["engine_upgrade_hint"] = {
            "newer_version": newer_incompatible[0].version_no,
            "required_engine": newer_incompatible[0].min_engine_version,
        }
    return result
```

## 6.4 维护流程

### 谁维护兼容性矩阵

| 角色 | 职责 |
|------|------|
| 地图数据团队 | 发布新格式版本时，声明 `breaking_change` 标志 |
| 引擎开发团队 | 发布新引擎版本时，声明 `min_data_format` / `max_data_format` |
| 平台运维团队 | 维护 `compatibility_matrix` 表，记录实测兼容结果 |

### 触发矩阵更新的3个时机

1. **新引擎版本发布**：新增一行，填写与所有现有格式版本的兼容性
2. **新数据格式版本发布**：新增一列，填写与所有现有引擎版本的兼容性
3. **发现兼容性问题**：更新对应行的 `known_issues`，必要时降级 `compatibility_level`

## 6.5 引擎与数据协调升级：三步法

**三步法**确保引擎升级过程中数据始终可用：

```
Step 1: 发布新格式数据，同时保留旧格式数据（两套并行）
Step 2: 推送引擎 OTA 升级，新引擎支持新旧两种格式
Step 3: 旧格式数据版本进入 DEPRECATED，最终下线
```

### 兼容窗口时间线

```
时间轴 ──────────────────────────────────────────────────────>

         T0            T1            T2            T3
         │             │             │             │
format_v3.5: PUBLISHED │             │  DEPRECATED │  清理
         │─────────────│─────────────│─────────────│
format_v4.0:           │  PUBLISHED  │             │ 长期保留
                       │─────────────│─────────────│──────────

                       │             │─────────────│

                       │─────────────│─────────────│──────────

         ← 兼容窗口 →  ← 过渡期 T1-T2 →
           两种格式均有效   引擎可选择
```
