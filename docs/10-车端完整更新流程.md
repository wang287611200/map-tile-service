## 12.2 在线/离线降级策略

### 网络良好时

```
优先使用在线实时瓦片（CDN 直连）
    ↓
后台静默下载新版本离线包（WiFi 优先）
    ↓
离线包下载完成后，切换离线模式使用新版本
```

### 网络不可用时（降级策略）

```
尝试在线请求 → 超时/失败
    ↓
检查本地离线包是否覆盖当前区域
    ├── 有离线包 → 使用离线数据（标记 cache_status=DEGRADED）
    │   ├── 离线包版本为最新 → 正常渲染
    │   └── 离线包版本较旧 → 提示用户"地图数据可能不是最新"
    └── 无离线包 → 使用过期缓存（EXPIRED 状态）
        └── 显示警告：地图数据可能已过期，建议联网更新
```

### 降级状态机

| 状态 | 触发条件 | 数据来源 | 用户提示 |
|------|----------|----------|----------|
| `ONLINE` | 网络正常 | CDN 实时瓦片 | 无 |
| `DEGRADED_OFFLINE` | 网络不可用 + 有离线包 | 本地离线包 | "已切换离线模式" |
| `DEGRADED_CACHE` | 网络不可用 + 无离线包 + 有过期缓存 | 本地缓存（EXPIRED） | "使用缓存地图，数据可能已过期" |
| `NO_MAP` | 以上都无 | — | "无可用地图数据，请联网" |

## 12.3 端到端时序图

```
车端                  CDN              瓦片服务           MQTT Broker
  │                    │                   │                   │
  │ ①启动，拉版本清单   │                   │                   │
  │────────────────────────────────────>  │                   │
  │                    │                   │ 查DB,返回清单     │
  │ <──────────────────────────────────── │                   │
  │ 比对本地版本，发现差异（CN_BJ 有更新）  │                   │
  │                    │                   │                   │
  │ ②预拉取前方瓦片（新版本URL）           │                   │
  │ GET /tiles/.../v20260101/14/13708/6411.pbf               │
  │────────────────>   │                   │                   │
  │                    │ CDN 缓存未命中     │                   │
  │                    │─────────────────> │（回源 OSS）       │
  │                    │ <───────────────── │                   │
  │ <──────────────── │                   │                   │
  │ 缓存新瓦片         │                   │                   │
  │                    │                   │                   │
  │ ③行驶中，实时请求当前视口瓦片           │                   │
  │────────────────>   │                   │                   │
  │ <──────────────── │ (CDN 命中，<10ms)  │                   │
  │                    │                   │                   │
  │                    │ ④服务端发布新版本  │                   │
  │                    │                   │───────────────>   │ PUBLISH
  │                    │                   │                   │
  │ ⑤收到版本变更推送  │                   │                   │
  │ <──────────────────────────────────────────────────────── │
  │ 标记 CN_BJ 区域瓦片为 STALE            │                   │
  │                    │                   │                   │
  │ ⑥下次请求 STALE 瓦片                  │                   │
  │ GET /tiles/.../v20260101/...           │                   │
  │────────────────>   │                   │                   │
  │ <──────────────── │ (版本化URL，正常响应)│                   │
  │ 更新缓存状态为 FRESH │                  │                   │
  │                    │                   │                   │
  │ ⑦304 优化（清单未变化）                │                   │
  │ GET /manifest (If-None-Match: hash)    │                   │
  │────────────────────────────────────>  │                   │
  │ <────────────────────────────────────  │ 304 Not Modified  │
  │ 无需处理                               │                   │
  │                    │                   │                   │
  │ ⑧进入隧道（网络断开），降级到离线模式  │                   │
  │ 使用本地 FRESH/DEGRADED 状态瓦片       │                   │
  │ 出隧道后重新上线，恢复在线模式          │                   │
  │                    │                   │                   │
```

### 关键时序说明

| 步骤 | 说明 | 延迟目标 |
|------|------|----------|
| ①版本清单拉取 | 启动时或定期轮询，支持 304 优化 | < 200ms |
| ②预拉取 | 后台预拉取，不阻塞导航 | 异步，不影响用户 |
| ③实时请求 | 行驶中的瓦片请求，CDN 高命中率 | < 50ms（CDN 命中） |
| ④-⑤ 推送 | MQTT 推送，秒级到达 | < 3s（MQTT 传输） |
| ⑥STALE 处理 | 收到推送后更新缓存状态，下次请求自动使用新版本 | 透明，用户无感知 |
| ⑦304 优化 | 清单未变化时，服务端返回 304，节省带宽 | < 50ms |
| ⑧降级 | 网络断开时无缝切换离线模式 | < 100ms |
