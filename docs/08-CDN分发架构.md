# 第8章 CDN 分发架构

---

## 10.1 双链路设计

```
┌────────────────────────────────────────────────────────────────────┐
│                        车载终端                                     │
│                                                                    │
│  ┌──────────────────────┐    ┌──────────────────────┐             │
│  │    链路① 版本协商     │    │    链路② 瓦片下载     │             │
│  │  (低频 / 动态)       │    │  (高频 / 静态)        │             │
│  └──────────┬───────────┘    └──────────┬────────────┘             │
└─────────────│──────────────────────────│────────────────────────-─┘
              │                           │
              ▼                           ▼
┌─────────────────────┐      ┌────────────────────────────────────┐
│   API Gateway       │      │          CDN 边缘节点               │
│  (业务逻辑处理)      │      │   cdn.example.com                  │
│                     │      │                                    │
│  版本协商服务         │      │  Cache-Control: immutable, 30d    │
│  灰度判断            │      │  命中率 > 95%                      │
│  兼容性矩阵查询       │      │                                    │
└──────────┬──────────┘      └──────────────┬─────────────────────┘
           │                                │ 回源（缓存未命中）
           │                                ▼
           │                 ┌────────────────────────────────────┐
           │                 │       OSS / S3 源站                 │
           └────────────────>│                                    │
                             │  静态文件（瓦片 + 离线包）            │
                             │  原生 ETag 支持                     │
                             └────────────────────────────────────┘
```

### 两条链路职责对比

| 对比维度 | 链路① 版本协商 | 链路② 瓦片下载 |
|----------|----------------|----------------|
| **请求目标** | API Gateway / 版本协商服务 | CDN → OSS |
| **请求频率** | 低频（启动时 + 定期轮询） | 高频（导航中每秒多次） |
| **内容类型** | JSON 动态响应 | 二进制 PBF 文件 |
| **是否需要业务逻辑** | 是（灰度判断、兼容性查询） | 否（纯文件下载） |
| **缓存策略** | `max-age=300`（5分钟） | `max-age=2592000, immutable`（30天） |
| **CDN 角色** | 不经过 CDN（直连 API Gateway） | 核心加速链路 |

## 10.2 版本号编码在URL路径中

### 三种方案对比

| 方案 | URL 示例 | 推荐度 | 说明 |
|------|----------|--------|------|
| **路径中（推荐）✅** | `/tiles/road_network/v20260101/14/13708/6411.pbf` | ✅ 强烈推荐 | URL 唯一，CDN 可永久缓存 |
| **查询参数（不推荐）❌** | `/tiles/14/13708/6411.pbf?version=v20260101` | ❌ 不推荐 | 许多 CDN 默认忽略查询参数，缓存混乱 |
| **业务服务路由（不推荐）❌** | `/tiles/14/13708/6411.pbf`（由服务端决定版本） | ❌ 不推荐 | 无法使用 CDN 静态缓存，每次都回源 |

### 核心思路转变

> **"缓存失效"问题 → "版本发现"问题**

- 传统方式：同一 URL 可能返回不同内容，必须通过 `Cache-Control` 控制缓存有效期，存在缓存污染风险
- 版本化 URL 方式：每个版本对应唯一 URL，内容永远不变，CDN 可以使用 `immutable` 无限期缓存
- 版本更新时：车端通过**版本协商**接口发现新版本号 → 自动生成新 URL → 请求新 URL → CDN miss → 回源
- 旧版本 URL 的缓存**自然淘汰**（30天后过期），无需主动清除

## 10.3 CDN 回源策略

### 三种方案对比

| 方案 | 架构 | 优点 | 缺点 | 推荐度 |
|------|------|------|------|--------|
| **方案A** | CDN → OSS 直连 | 简单，OSS 原生 ETag | CDN 无法动态鉴权 | 部分场景 |
| **方案B** | CDN → 业务服务 | 可动态鉴权、灵活路由 | 每次回源都经过业务服务，高并发压力大 | ❌ 不推荐 |
| **方案C** | CDN → OSS（主）+ CDN → 业务服务（鉴权） ✅ | 静态文件直连 OSS，鉴权由独立服务处理 | 架构稍复杂 | ✅ 推荐 |

**混合方案（推荐）**：
- 公开版本瓦片：CDN → OSS 直连（无鉴权，适合非敏感地图数据）
- 高精度/付费版本：CDN 回源时先请求鉴权服务生成签名 URL，再从 OSS 获取

### CDN 条件请求两种模式

```
模式1：缓存命中
  车端 → CDN（有缓存）
    CDN 直接返回，不回源
    响应时间：< 10ms（边缘节点）

模式2：缓存未命中（回源）
  车端 → CDN（无缓存）
    CDN → OSS（发送 If-None-Match: "prev_etag"）
    ├─ OSS 返回 304：CDN 刷新缓存TTL，返回 304 给车端（极少发生，因版本化URL内容不变）
    └─ OSS 返回 200：CDN 缓存新文件，返回给车端
    响应时间：CDN 到 OSS < 5ms（骨干网），总计 < 50ms
```

## 10.4 OSS/S3 原生 ETag 支持

OSS 和 S3 在文件上传时会**自动计算 MD5 ETag**，无需任何额外代码。

### 工作原理

```
上传时：
  PUT /road_network/v20260101/14/13708/6411.pbf
  → OSS 自动计算文件 MD5，存储为 ETag

回源时（CDN → OSS）：
  GET /road_network/v20260101/14/13708/6411.pbf
  If-None-Match: "prev_etag"
  → OSS 比较 ETag，相同则返回 304，不同则返回 200 + 新文件
```

### 关键洞察

> 在版本化 URL + `immutable` 的架构下，**ETag 不再是缓存新鲜度的核心手段**。
> 因为版本化 URL 保证了同一 URL 的内容永远不变，`ETag` 在此场景下降级为：
> - CDN 内部节点间同步的一致性校验
> - 极端情况下（CDN 配置错误）的兜底机制

## 10.5 缓存控制策略

| 资源类型 | Cache-Control 策略 | 有效期 | 说明 |
|----------|-------------------|--------|------|
| **瓦片文件** | `public, max-age=2592000, immutable` | **30天** | 版本化URL，内容永不变，CDN 可无限期缓存 |
| **版本清单** | `public, max-age=300` | **5分钟** | 需要及时感知版本变化 |
| **离线包** | `public, max-age=2592000, immutable` | **30天** | 同瓦片文件，版本化URL |
| **版本协商响应** | `private, no-store` | **不缓存** | 含车辆个性化信息（灰度分桶），不可缓存 |

### 缓存分层架构

```
车端本地缓存
    ↑ Cache-Control: immutable（瓦片永久缓存）
CDN 边缘节点
    ↑ Cache-Control: max-age=2592000（30天）
CDN 回源层
    ↑ If-None-Match / ETag（内容一致性校验）
OSS 源站
    永久存储（版本文件从不被覆盖）
```
